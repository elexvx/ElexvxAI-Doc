## 背景

在一个“文档站点”里，通常同时存在两类内容：

- **文档（Docs）**：稳定、结构化、可版本化（例如 v1/v2）；
- **博客（Blog）**：非线性、偏叙事、迭代快（公告、复盘、实践记录）。

将 Blog 与 Docs 分离后，你可以：

1. 给文档更清晰的信息架构（目录、版本、侧边栏）
2. 给博客更自然的时间线（按日期排序、标签聚合）
3. 在搜索层面做区分（Type=Docs/Blog）

## 内容复用建议

当一篇博客沉淀为“长期可维护”的信息后，建议迁移到文档：

| 内容类型 | 适合放在 | 原因 |
| --- | --- | --- |
| 快速公告 | Blog | 生命周期短 |
| API 约定 | Docs | 需要版本化 |
| 实践复盘 | Blog | 强叙事、可追溯 |
| 最佳实践 | Docs | 需要可索引与维护 |

## 示例：从博客迁移到文档

当你发现下面这种段落开始稳定重复出现：

```md
## 认证
所有请求都需要 Bearer token……
```

就把它迁移到 `docs/zh/v2/...`，并在博客里保留链接即可。
